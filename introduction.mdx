The Point Cloud Library (PCL) is a standalone, large-scale open-source project for 2D/3D image and point cloud processing. Released under the BSD license, PCL is free for both commercial and research use. Version **1.15.1** is the current stable release.

PCL is used across robotics, autonomous driving, computer vision, 3D scanning, augmented reality, and any domain where algorithms need to operate on spatial point data. It provides a unified C++17 API that covers the full processing pipeline — from raw sensor input through filtering, feature estimation, segmentation, registration, surface reconstruction, and interactive visualization.

PCL is organized into focused, independently linkable modules. Each module has a well-defined responsibility and a set of dependencies on other modules.

## Architecture

PCL follows a template-based architecture. Most algorithms are implemented as class templates parameterized on a point type `PointT`. This lets you instantiate only the combinations you need, keeping compile times and binary sizes manageable.

The central data structure is `pcl::PointCloud<PointT>`, defined in `common`. It stores a flat vector of points alongside metadata such as width, height, sensor origin, and sensor orientation. An _organized_ cloud (`height > 1`) preserves the 2D grid structure of a depth image; an _unorganized_ cloud (`height == 1`) is a flat list.

Algorithms receive input through `setInputCloud()` and write output through a `filter()`, `compute()`, or `align()` call. Most intermediate results are also `pcl::PointCloud<PointT>` objects, so modules compose naturally.

```
Sensor / file  →  io  →  filters  →  features / keypoints
                                   ↘  segmentation
                                   ↘  registration
                                   ↘  surface
                                   ↘  visualization
```
